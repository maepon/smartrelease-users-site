# 復習：SFTP接続とファイル転送の基本コマンド

SmartRelease Uのテストサーバーは、セキュリティの仕様上**SFTP（ファイル転送方式）のみ**が許可されており、通常のSSHシェル接続（リモートコマンド実行）は制限されています。
SCPやrsyncも利用できませんが、SFTPプロトコルを使用してファイルのアップロードやダウンロード、リモートディレクトリの管理が可能です。
本ドキュメントでは、接続を効率化するためのSSH設定ファイル（.ssh/config）の記述方法と、CUI環境での主要なSFTPコマンドについて解説します。

## 1\. .ssh/config の設定

毎回長いコマンド（ホスト名やユーザー名、鍵パスの指定）を入力する手間を省くため、.ssh/config にホスト設定（エイリアス）を定義します。

### 設定例

\~/.ssh/config に以下の記述を追加します（パーミッションは 600 である必要があります）。  
```
Host [エイリアス名（任意 例:smartrelease-test）]
    HostName ftp.pre-svr.jp
    User [FTPユーザー作成時に指定したユーザー名]
    Port 22
    IdentityFile ~/.ssh/[FTPユーザー作成時に生成された秘密鍵ファイル名]
    IdentitiesOnly yes
```

### 設定項目の補足

* **Host**: 任意の識別名（エイリアス）を指定します。上記例では smartrelease-test としています。  
* **IdentityFile**: FTPユーザー作成時に生成された秘密鍵ファイルを指定します。パーミッションは 600 に設定してください。
* **IdentitiesOnly**: yes を指定することで、指定した秘密鍵（IdentityFile）のみを確実に使用させ、鍵の不一致による接続拒否やロックアウトを防ぎます。

### 接続検証

設定完了後、以下の簡略化されたコマンドで接続可能になります。  
```
sftp smartrelease-test
```

## 2\. ファイル転送で利用する主要SFTPコマンド

SFTPプロンプト（ sftp\> ）に移行した後に使用する基本コマンドの一覧です。SFTPでは、**リモート（接続先サーバー）**と**ローカル（自身の開発環境）**の双方のディレクトリを意識して操作します。

### 2.1 ディレクトリ移動・確認

リモート側は通常のコマンド、ローカル側は頭に l (local) を付けたコマンドを使用します。

| 操作内容 | リモート側（サーバー） | ローカル側（手元）   |
| :---- | :---- | :---- |
| **現在地のパスを表示** | pwd | lpwd |
| **ファイル・ディレクトリ一覧** | ls または ls \-la | lls または lls \-la |
| **ディレクトリの移動** | cd \[ディレクトリ名\] | lcd \[ディレクトリ名\] |

### 2.2 ファイルのアップロード（ローカル → リモート）

手元のファイルをテストサーバーに転送します。

**単一ファイルのアップロード**  
```
sftp> put index.html
```

**ローカルでのファイル名を変更してアップロード**  
```
sftp> put index.html index_backup.html
```  
**ディレクトリごと再帰的にアップロード**  
```
sftp> put -r dist/
```

### 2.3 ファイルのダウンロード（リモート → ローカル）

テストサーバーのファイルを手元に取得します。

**単一ファイルのダウンロード**  
```
sftp> get remote_config.php
```  

**ディレクトリごと再帰的にダウンロード**  
```
sftp> get -r uploads/
```

### 2.4 リモート側でのファイル・ディレクトリ操作

シェル接続は不可ですが、SFTPプロンプト経由で限定的なファイル操作が可能です。

**ディレクトリの作成**  
```
sftp> mkdir new_images
```

**ファイルの削除**
```
sftp> rm old_file.txt
```

**ディレクトリの削除（空である必要があります）**  
```
sftp> rmdir old_dir
```

**ファイル名の変更・移動**  
```
sftp> rename old_name.php new_name.php
```

### 2.5 終了処理

**SFTPセッションを終了してローカルに戻る**  
```
sftp> bye
```
*(※ exit または quit でも同様に終了可能です)*
